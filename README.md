## 1. Модификации программы (относительно исходного кода)

1.1 **Автоматизация экспериментов**  
   Программа самостоятельно перебирает фиксированный набор размеров матриц:
200, 400, 800, 1200, 1600, 2000, 2400, 2800, 3200.

Для каждого размера выполняется умножение, измеряется время выполнения, и результаты сохраняются в файлы. Это позволяет провести систематическое исследование производительности без ручного перезапуска.

1.2. **Файловый вывод с привязкой к размеру**  
  Исходные матрицы A и B, а также результирующая матрица C сохраняются в отдельные файлы с именем, содержащим размерность:

A_<N>.txt — первая матрица

B_<N>.txt — вторая матрица

C_<N>.txt — результат умножения

Такой подход исключает путаницу при обработке разных N и позволяет легко сопоставить результаты различных запусков.
1.3. **Повышение точности вывода**  
 При записи матриц в файлы используется фиксированный формат с точностью 12 знаков после запятой (fixed << setprecision(12)). Это обеспечивает сохранение всех значащих цифр для последующей верификации.

1.4. **Инициализация матриц**  
В отличие от исходного кода, где матрицы заполнялись по формулам, здесь используется генератор случайных чисел (uniform_real_distribution). Это делает тесты более разнообразными и позволяет проверить корректность умножения на случайных данных.

1.5. **Автоматическая верификация на Python**
Разработан Python-скрипт, который:
считывает все сгенерированные файлы матриц;
вычисляет эталонное произведение с помощью библиотеки NumPy (с двойной точностью);
сравнивает с результатами программы и вычисляет:
максимальную абсолютную ошибку;
среднюю абсолютную ошибку;
максимальную относительную ошибку;
формирует сводную таблицу и сохраняет её в CSV-файл.





## 2. Ход работы

### 2.1. Создание программы
На языке C++ написана программа, реализующая классический алгоритм последовательного умножения квадратных матриц с тремя вложенными циклами. Сложность алгоритма — O(N³). Для измерения времени выполнения используется библиотека std::chrono::high_resolution_clock. Программа автоматически проходит по всем заданным размерам, генерирует случайные матрицы, выполняет умножение, сохраняет результаты и время выполнения.

### 2.2. Проведение экспериментов 
Для каждого размера N из списка получены:
файлы матриц A_N.txt, B_N.txt, C_N.txt;
время выполнения (записано в timings.txt).

### 2.3. Верификация
Запущен Python-скрипт verify.py, который обработал все пары файлов и вычислил статистику погрешностей.
